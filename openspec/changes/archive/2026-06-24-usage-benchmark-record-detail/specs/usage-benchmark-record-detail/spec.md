## ADDED Requirements

### Requirement: 测算记录可下钻打开详情

看板「用量测算」历史记录列表 SHALL 支持点击任意一条已结束的记录打开「记录详情」面板。详情面板 MUST 沿用现有 workspace 详情的抽屉(`ElDrawer`)交互模式,展示该记录的元信息(标签 / 时间窗 / 时长 / 各来源与跨来源 token 合计)。点击进入详情 MUST NOT 与既有「对比勾选」交互冲突——勾选对比的复选框点击不触发打开详情。

#### Scenario: 点击记录打开详情

- **WHEN** 用户在历史记录列表点击一条已结束记录的卡片主体
- **THEN** 系统打开记录详情面板,展示该记录的标签、时间窗、时长与各来源/合计 token 数据

#### Scenario: 勾选对比不打开详情

- **WHEN** 用户点击记录卡片上的「对比」复选框
- **THEN** 系统仅切换该记录的对比选中态,不打开记录详情面板

#### Scenario: 关闭详情

- **WHEN** 用户在记录详情面板触发关闭
- **THEN** 面板关闭,历史记录列表保持原有筛选/对比状态不变

### Requirement: 记录详情展示窗口内各会话实际消耗

记录详情面板 SHALL 展示该测算记录窗口内涉及的各会话的实际消耗。会话集合 MUST 由记录已落盘的各来源去重 sessionId 列表(`totals[source].sessionIds`)汇总而成,并按会话 key(`${source}:${sessionId}`)反查会话维度用量记录得到明细。每个可解析的会话 MUST 复用「会话用量明细」的会话行渲染(会话标题、工具与 model、项目与分支、时间窗、对话轮次、用量条与 token 合计),其中 jiraKey 徽标的下钻行为与既有会话明细一致。会话列表 SHALL 默认按会话有效用量合计(total)倒序展示。

#### Scenario: 展示窗口内各会话明细

- **WHEN** 用户打开一条记录的详情,该记录 `totals` 中各来源的 `sessionIds` 能在会话维度 store 中反查到对应记录
- **THEN** 详情面板按 total 倒序逐行展示这些会话的标题、工具·model、项目·分支、时间窗、轮次与用量条

#### Scenario: 复用会话明细行渲染

- **WHEN** 详情面板渲染某个会话行
- **THEN** 其展示形态与「AI 用量」会话用量明细中的会话行一致(同一可复用组件),含 jiraKey 徽标可下钻需求详情

#### Scenario: 多来源会话合并展示

- **WHEN** 某记录窗口跨越多个来源(如 cursor 与 codex)且各自有 sessionId
- **THEN** 详情面板把各来源会话合并到同一列表中,按 total 倒序统一排序展示

### Requirement: 记录详情会话用量口径说明与兜底

记录详情面板 SHALL 明确标注所展示的会话 token 为「会话累计」口径(会话生命周期 firstAt→lastAt 的累计),并提示其可能与测算窗口内的合计不完全一致(会话可能在窗口之外仍有活动)。对记录中存在、但在会话维度 store 中无法解析的 sessionId(采集开关在窗口期间曾关闭、或记录已被会话维度保留上限裁剪),系统 MUST 安全兜底:以计数或提示文案告知用户有 N 个会话无可用明细,MUST NOT 报错或中断面板渲染。当记录的全部 sessionId 均不可解析时,详情面板 SHALL 呈现解释性空态。

#### Scenario: 标注会话累计口径

- **WHEN** 详情面板展示会话用量明细
- **THEN** 面板含说明文案,表明会话 token 为会话累计口径,可能与测算窗口内合计存在差异

#### Scenario: 部分 sessionId 无法解析

- **WHEN** 记录的某些 sessionId 在会话维度 store 中查不到对应记录
- **THEN** 详情面板展示可解析的会话明细,并以计数/提示告知存在 N 个无明细会话,不报错

#### Scenario: 全部不可解析时空态

- **WHEN** 记录的全部 sessionId 均无法在会话维度 store 中解析
- **THEN** 详情面板呈现解释性空态文案(如采集开关曾关闭导致无会话明细),不报错
