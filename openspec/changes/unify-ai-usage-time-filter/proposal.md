## Why

「AI 用量」页面当前有三个模块各自为政的时间维度:汇总卡片固定只看「当天」、用量趋势图硬编码「近 14 天」、会话用量明细有自己独立的「时间范围」下拉(当天 / 近 7 天 / 近 30 天)。用户切换会话明细的时间范围时,上方卡片与趋势图纹丝不动,导致同一页面三块数据落在不同时间口径上,既难对照又造成认知割裂。需要一个统一的页面级时间筛选来一键对齐全页数据口径。

## What Changes

- 在「AI 用量」页面顶部新增**全页统一时间筛选**,三档:**当天 / 近 7 天 / 近 30 天**。
- 切换该筛选后,**同步驱动**页面内全部三个模块:
  - **AI 用量汇总卡片**:从「固定当天」改为「按所选时间范围聚合」的 token 与对话次数;卡片单位文案随范围动态变化(如「当天 token」/「近 7 天 token」)。
  - **用量趋势图**:从硬编码 `DAYS=14` 改为「按所选范围天数」绘制按日序列;图表副标题随范围动态变化。
  - **会话用量明细**:**移除**该模块自带的「时间范围」下拉,改为消费页面级统一筛选;其余筛选(AI 平台 / 所属项目 / 排序)保留。
- 统一筛选的选中值在本机持久化(localStorage),刷新页面后保持上次选择。
- **BREAKING**(交互层面,非数据层面):会话明细模块原有的独立「时间范围」下拉被移除,时间口径统一由顶部筛选托管。

## Capabilities

### New Capabilities

(无新增 capability,本变更是对既有「AI 用量」页面交互的统一收敛。)

### Modified Capabilities

- `ai-usage-overview`: 新增「页面级统一时间筛选」要求;并修改「AI 用量看板页面」要求,使汇总卡片改为按所选范围聚合、趋势图改为按所选范围天数绘制(原「卡片固定展示当天」「趋势图固定近 N 天」表述随之调整)。
- `session-usage-list-display`: 修改「会话用量列表筛选与排序」要求,移除会话模块自带的「时间范围」下拉,改由页面级统一筛选提供时间口径(查询端点的 `from` 时间参数语义不变,仅由统一筛选驱动)。

## Impact

- 前端:`packages/ui/src/tabs/AiUsageTab.vue`(唯一承载三模块的页面组件)——新增统一时间筛选 state(替代 `DAYS` 常量与 `sessionRangeDays` 本地 ref)、卡片聚合逻辑、趋势图与会话查询的参数联动、模板中顶部筛选 UI 与会话模块下拉删除。
- API 层:沿用既有 `fetchAiUsage(days)` 与 `fetchSessionUsage({ from })`,**无需新增端点**;`fetchAiUsage` 的 `days` 入参由统一筛选驱动。
- 后端:`GET /ai-productivity/ai-usage?days=N` 与 `GET /ai-productivity/session-usage?from=` **行为不变**,无需改动。
- 数据 schema / 采集链路:不涉及。
- 持久化:新增一个 localStorage 偏好键存储统一筛选选中值。
